🔍 核心問題分析

操作不易

高優先級

三欄佈局太擁擠，默認寬度不足，導致交互困難。按鈕和控制項太小，難以點擊。

  • ✗ 左側欄寬度僅 240px，分支名稱被截斷
  • ✗ 右側欄寬度僅 320px，文件列表擁擠
  • ✗ 按鈕高度不足 48px，不符合 Fitts 定律
  • ✗ Checkbox 僅 14x14px，難以精準點擊

信息無法正確看到內文

高優先級

差異查看器、文件列表、提交信息等關鍵內容因空間不足而被截斷。

  • ✗ 差異查看器行號佔用 80px (40px × 2)，代碼區域被壓縮
  • ✗ 文件名使用 text-overflow: ellipsis 但無 tooltip
  • ✗ 提交信息單行顯示，多行內容被隱藏
  • ✗ 代碼長行無法換行，需要水平滾動

響應式斷點定義

窗口寬度 左欄 右欄 行號寬度 字體大小 策略
< 1200px Tab 切換 / 可折疊 / 抽屜 32px × 2 11px 彈性佈局
1200-1600px 240px 280px 36px × 2 12px 標準模式
> 1600px 300px 360px 36px × 2 13px 展開模式

佈局模式切換邏輯

Tab 切換模式
• 左側欄變為垂直 Tab 列
• 點擊 Tab 切換顯示分支/提交/文件
• 中央區域最大化
• 適用場景：筆記型電腦、小屏幕
可折疊側欄模式
• 左側欄可收合為圖標列
• 懸停或點擊展開完整內容
• 保持三欄結構但更靈活
• 適用場景：需要快速切換的開發者
單欄 + 抽屜模式
• 默認只顯示中央工作區域
• 右側欄變為抽屜，點擊按鈕展開
• 最大化代碼查看空間
• 適用場景：專注於代碼審查
標準三欄模式
• 保持現有三欄佈局
• 根據窗口大小調整欄寬
• 適用場景：大屏幕、外接顯示器

CSS 實現方案

+page.svelte (Svelte)
// 響應式寬度計算
let sidebarWidth = 240;
let rightWidth = 320;

// 根據窗口大小選擇佈局模式
function updateLayout() {
  const w = window.innerWidth;
  if (w < 1200) {
    sidebarWidth = 180;
    rightWidth = 200;
  } else if (w < 1600) {
    sidebarWidth = 240;
    rightWidth = 280;
  } else {
    sidebarWidth = 300;
    rightWidth = 360;
  }
}
